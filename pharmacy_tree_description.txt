ДЕТАЛЬНОЕ ОПИСАНИЕ УЗЛОВ ДИАГРАММЫ
═══════════════════════════════════════════════════════════════════════════════

УРОВЕНЬ 1: ОСНОВНЫЕ ТЕХНИЧЕСКИЕ ПРОБЛЕМЫ
─────────────────────────────────────────

┌─ Интеграция платежных систем
│  Проблема: Сложность интеграции с ЮKassa, Сбербанк, обработка платежей
│  Влияние: Критическая задержка 10-15 дней
│  Причины: Сложный API, требования безопасности, тестирование транзакций
│
└─ Сложность API
   Проблема: Сложные и нестабильные API внешних сервисов
   Влияние: Задержка 5-10 дней
   Причины: Плохая документация, изменения версий, отладка


УРОВЕНЬ 2: СПЕЦИФИЧЕСКИЕ ЗАДАЧИ
────────────────────────────────

┌─ Работа с электронными рецептами
│  Проблема: Распознавание, валидация, хранение медицинских документов
│  Влияние: Задержка 15-20 дней
│  Причины: Сложность OCR, требования законодательства, интеграция с ЕГИСЗ
│
└─ Человеческий фактор
   Проблема: Ошибки, неправильные оценки, коммуникационные проблемы
   Влияние: Задержка 10-15 дней
   Причины: Субъективные оценки, недопонимание требований


УРОВЕНЬ 3: ИНТЕГРАЦИОННЫЕ И КАДРОВЫЕ ПРОБЛЕМЫ
───────────────────────────────────────────────

┌─ Интеграция с ЕГИСЗ
│  Проблема: Подключение к государственной системе здравоохранения
│  Влияние: Задержка 10-15 дней
│  Причины: Бюрократия, сложная процедура получения доступа, нестабильное API
│
├─ Валидация медицинских документов
│  Проблема: Проверка подлинности и корректности рецептов
│  Влияние: Задержка 7-10 дней
│  Причины: Разные форматы рецептов, требования валидации
│
├─ Выгорание команды разработки
│  Проблема: Потеря продуктивности из-за переработок
│  Влияние: Задержка 15-20 дней
│  Причины: Сжатые сроки, высокая нагрузка, монотонность задач
│
└─ Низкая мотивация
   Проблема: Снижение эффективности команды
   Влияние: Задержка 10-12 дней
   Причины: Неясные цели, отсутствие обратной связи, проблемы с менеджментом


УРОВЕНЬ 4: БАЗОВЫЕ ПРИЧИНЫ
───────────────────────────

┌─ Дефицит кадров
│  Проблема: Нехватка специалистов на рынке труда
│  Влияние: Задержка 20-30 дней
│  Причины: Высокий спрос на разработчиков, специфические навыки (медицина + IT)
│
├─ Затяжные согласования
│  Проблема: Долгие процессы согласования решений
│  Влияние: Задержка 10-15 дней
│  Причины: Многоуровневая иерархия, отсутствие полномочий у команды
│
├─ Недостаточная квалификация
│  Проблема: Команде не хватает экспертизы в фармацевтической отрасли
│  Влияние: Задержка 15-20 дней
│  Причины: Нет опыта в медицинских системах, незнание законодательства
│
└─ Сложность архитектуры системы
   Проблема: Большое количество интеграций и бизнес-логики
   Влияние: Задержка 10-15 дней
   Причины: Микросервисная архитектура, множество внешних API


УРОВЕНЬ 5: КОРНЕВАЯ ПРИЧИНА
────────────────────────────

└─ Высокая нагрузка на ключевых специалистов
   Проблема: Архитекторы и тимлиды перегружены задачами
   Влияние: Задержка 20-25 дней (критическая!)
   Причины: 
   • Все решения проходят через одного-двух специалистов
   • Недостаточное делегирование полномочий
   • Отсутствие документации и знаний у команды
   • Сложность системы требует постоянного участия экспертов


═══════════════════════════════════════════════════════════════════════════════
АНАЛИЗ ВЗАИМОСВЯЗЕЙ
═══════════════════════════════════════════════════════════════════════════════

Корневая причина "Высокая нагрузка на ключевых специалистов" порождает:
  ↓
  1. Затяжные согласования (специалисты не успевают)
  2. Недостаточную квалификацию команды (нет времени на обучение)
  3. Сложность системы (нет времени на упрощение архитектуры)
     ↓
     Это приводит к:
     ↓
     • Выгоранию команды (из-за согласований)
     • Низкой мотивации (нет автономности)
     • Дефициту кадров (сложно найти спецов)
        ↓
        Что вызывает:
        ↓
        • Проблемы с интеграцией ЕГИСЗ
        • Проблемы с валидацией документов
           ↓
           Итог: задержка всех ключевых задач проекта!

РЕКОМЕНДАЦИИ ПО УСТРАНЕНИЮ ЗАДЕРЖЕК
═══════════════════════════════════════════════════════════════════════════════

1. СНИЖЕНИЕ НАГРУЗКИ НА КЛЮЧЕВЫХ СПЕЦИАЛИСТОВ
   ✓ Делегирование полномочий senior разработчикам
   ✓ Создание подробной технической документации
   ✓ Автоматизация рутинных задач
   ✓ Найм дополнительного архитектора

2. УЛУЧШЕНИЕ ПРОЦЕССОВ СОГЛАСОВАНИЯ
   ✓ Использование Agile-подхода (короткие итерации)
   ✓ Предоставление полномочий команде
   ✓ Автоматизация через системы трекинга (Jira)

3. ПОВЫШЕНИЕ КВАЛИФИКАЦИИ КОМАНДЫ
   ✓ Обучение специфике фармацевтической отрасли
   ✓ Тренинги по работе с медицинскими системами
   ✓ Консультации с экспертами отрасли

4. УПРОЩЕНИЕ АРХИТЕКТУРЫ
   ✓ Разделение системы на независимые модули
   ✓ Использование готовых решений где возможно
   ✓ Создание абстракций для сложных интеграций

5. ЗАБОТА О КОМАНДЕ
   ✓ Адекватное планирование нагрузки
   ✓ Регулярные перерывы и отпуска
   ✓ Team building активности
   ✓ Прозрачная система мотивации

ОЖИДАЕМЫЙ ЭФФЕКТ ОТ УСТРАНЕНИЯ ПРИЧИН
═══════════════════════════════════════════════════════════════════════════════

При устранении корневой причины и следовании рекомендациям:

  • Сокращение задержек на 50-60%
  • Повышение производительности команды на 30-40%
  • Снижение рисков срыва сроков
  • Улучшение качества кода и архитектуры
  • Повышение удовлетворенности команды
