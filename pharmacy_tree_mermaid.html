Выявленные категории причин задержек:

  • Организационные риски:
    • Затяжные согласования требований и изменений
    • Дефицит квалифицированных кадров на рынке
    • Высокая нагрузка на ключевых специалистов (архитекторов, тимлидов)
  • Технические риски:
    • Сложность интеграции с платежными системами (ЮKassa, Сбербанк)
    • Интеграция с ЕГИСЗ (Единая государственная информационная система здравоохранения)
    • Работа с электронными рецептами и медицинской документацией
    • Сложность API внешних систем
  • Кадровые риски:
    • Выгорание команды разработки при длительных проектах
    • Недостаточная квалификация в специфических областях (медицина, фармацевтика)
    • Низкая мотивация команды
  • Процессные риски:
    • Валидация электронных рецептов и медицинских документов
    • Необходимость соблюдения регуляторных требований (152-ФЗ, приказы Минздрава)
  • Системные риски:
    • Сложность общей архитектуры системы
    • Человеческий фактор в процессе разработки

Рекомендации по минимизации рисков:

  • ✓ Привлечение консультантов по медицинским системам на ранних этапах
  • ✓ Раннее начало работы с интеграциями (ЕГИСЗ, платежные системы)
  • ✓ Создание прототипов для валидации сложных требований
  • ✓ Регулярные ретроспективы и забота о команде для предотвращения выгорания
  • ✓ Обучение команды специфике фармацевтической отрасли
  • ✓ Автоматизация процессов согласования через системы трекинга
